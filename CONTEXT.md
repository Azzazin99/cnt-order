# Domain Glossary

## Login

ทุกบัญชีเข้าสู่ระบบที่ `/` ด้วยฟอร์มเดียว (ชื่อผู้ใช้ + รหัสผ่าน) — UI ไม่แยกประเภทบัญชี เพราะ backend ใช้ `username` เป็นตัวแบ่งบทบาท: รหัส SMIS 8 หลัก → School Account, ชื่อผู้ใช้แอดมินเขต → Platform Admin

## Platform Admin

ผู้ดูแลระดับเขตพื้นที่การศึกษา ที่ login ด้วยชื่อผู้ใช้แอดมินเขต (เช่น `admin`) และมี `schoolId = null` ใน session — เห็นและจัดการเอกสารทุกโรงเรียน รวมถึงตั้งค่าโรงเรียนและรหัสโรงเรียน

## School Account

บัญชีร่วมของโรงเรียนหนึ่งแห่ง ใช้ `username` และ `password` ที่อ้างอิงรหัส SMIS มี `schoolId` ผูกกับโรงเรียนนั้น — เห็นและจัดการได้เฉพาะเอกสารของโรงเรียนตัวเองใน `/admin`

## School Code

รหัส SMIS 8 หลักของโรงเรียน ใช้เป็นทั้ง username และรหัสผ่านเริ่มต้นของ School Account

## MOE Code

รหัสกระทรวง 10 หลักของโรงเรียน เก็บเป็นข้อมูลอ้างอิงในระบบ ไม่ใช้สำหรับ login

## Document

คำสั่งหรือเอกสาร PDF ที่อัปโหลดในระบบ แต่ละรายการผูกกับ `schoolId` ของโรงเรียนที่อัปโหลด (หรือ `null` สำหรับเอกสารระดับเขตที่แสดงบนหน้าสาธารณะ) และมี **หมวดหมู่คำสั่ง** บังคับเลือกตอนเพิ่ม/แก้ไข

## Document Organization (หน่วยงาน)

ฟิลด์ "หน่วยงาน" ในฟอร์มเพิ่ม/แก้ไขเอกสารที่ `/admin` คือการเลือก `schoolId` ของเอกสาร:

- **Platform Admin** — เลือกได้ระหว่าง **สพป.ชัยนาท** (`schoolId = null`) หรือโรงเรียนใดโรงเรียนหนึ่งในระบบ (156 โรงเรียนจาก `_สพป.ชัยนาท.xlsx`)
- **School Account** — บังคับเป็นชื่อโรงเรียนที่ login อยู่ (อ่านอย่างเดียว) ระบบใช้ `schoolId` จาก session ฝั่ง API ไม่รับค่าจากฟอร์ม

## Document Category

ประเภทคำสั่ง fixed list 7 รายการ ใช้แทนฟิลด์กลุ่มงานเดิม:

- เลื่อน/แก้ไข เงินเดือน (`salary_adjustment`)
- บรรจุ/ประเมิน ครูผู้ช่วย (`assistant_appointment`)
- แต่งตั้งครูผู้ช่วยให้ดำรงตำแหน่งครู (`assistant_to_teacher`)
- ย้าย/โอน ข้าราชการครู (`teacher_transfer`)
- วิทยฐานะ (`academic_rank`)
- ลาออก (`resignation`)
- อื่น ๆ (`others`) — ค่าเริ่มต้นสำหรับเอกสารเก่าที่ migrate

## School

หน่วยงานโรงเรียนในระบบ มีชื่อ รหัส SMIS รหัสกระทรวง (อ้างอิง) และสถานะ active/inactive
